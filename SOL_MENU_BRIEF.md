# SOL Menu — Session Brief

## What this project is

A build pipeline for a self-contained luxury restaurant menu HTML file (`SOL_v3.html`). The goal: edit data in Airtable → run one command → get a fresh `SOL_v3_built.html` with baked-in data → deploy.

## How the build works

`build.py` reads `SOL_v3.html`, uses **regex** to find and replace four data blocks, then writes `SOL_v3_built.html`. The four blocks it targets:

```
const SOL_DATA = { ... };    ← restaurant meta (phone, email, hours, socials)
const ITEMS=[...];           ← 104 menu dishes
const BBQ={...};             ← BBQ cuts by tier + assortments
const SIG=[...];             ← Chef's Path tasting menus
```

Regex confirmed working — all four blocks found in the real HTML file.

## Airtable — 6 tables required

See `airtable_schema.md` for full field definitions. Summary:

| Table | What it holds |
|---|---|
| `Menu Items` | 104 dishes — section, Ko, Rom, En, Description, Prices, Tags, Sub, Sort |
| `BBQ Cuts` | Individual cuts — Ko, En, Tier (premium/regular/duck), Price, Highlight |
| `BBQ Assortments` | Pre-set combos — Name, Items (one per line), Price, Highlight, Sort |
| `Tasting` | Chef's Path menus — Title, Description, Courses (Ko · En per line), Price, Sort |
| `Meta` | Key/Value pairs — phone, whatsapp, email, instagram, facebook, address, ga4, hours |
| `Photos` | Dish Name → URL mapping (optional, leave empty to start) |

### Prices field format (Menu Items)
- Single price: `1350`
- Multi-variant: `3 pcs:1350, 5 pcs:2150`

### Tasting Courses field format
One dish per line:
```
게 새우 샐러드 · Crab Prawn Salad
꿀새우볼 · Honey Shrimp Ball
```

## GitHub Actions

Workflow at `.github/workflows/build.yml`:
- **Manual trigger**: Actions tab → "Build SOL Menu" → Run workflow
- **Auto trigger**: fires when `SOL_v3.html` is pushed
- Add `AIRTABLE_TOKEN` and `AIRTABLE_BASE_ID` as repo secrets (Settings → Secrets → Actions)
- Built file is uploaded as a downloadable artifact and committed back to the repo

## Key constraints (do NOT change in HTML)

- No optional chaining `?.`
- No `URLSearchParams`
- No `const`/`let` inside swipe handlers
- Single self-contained HTML — no external JS dependencies
- Build always writes to `SOL_v3_built.html`, never touches `SOL_v3.html`
